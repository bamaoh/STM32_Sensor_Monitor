# STM32 Sensor Monitor

## Overview
This project is an atmospheric sensor monitoring system built on FreeRTOS with a layered, AUTOSAR-like architecture. It reads sensor data, applies filtering, manages system states, and performs fault diagnostics.

## Hardware
* **MCU Board:** STM32 Nucleo-F446RE (STM32F446RET6/64pin/512k Flash/128k Ram/180MHz Freq)
* **Sensor:** BME280 Atmospheric Sensor (Temperature, Humidity, Pressure, SPI/I2C)

## Software Architecture
The software is structured into three main layers:

* **ASW (Application Software):**
  * `Asw_Sensor`: Reads sensor data via RTE, applies compensation calculation, moving average filter, and unit conversion.
  * `Asw_Manage`: Manages the system state machine (INIT → NORMAL → FAULT).
  * `Asw_Diag`: Diagnoses sensor timeouts and data validity.
* **RTE (Runtime Environment):**
  * `Rte_App`: Data exchange interface between ASW and BSW. Provides read/write access to shared data buffers (e.g., sensor values, diagnostic flags, system state). Both ASW and BSW only interact through this data buffer — neither calls the other's functions directly.
  * `Rte_Hal`: Function call interface reserved for cases where ASW needs to directly trigger BSW operations (e.g., force sensor re-read, system reset). Not used in normal periodic data exchange.
* **BSW (Base Software):**
  * **MCAL:** Not implemented manually. The STM32 HAL library (auto-generated by CubeMX under `Core/` and `Drivers/`) serves as the MCAL, providing direct hardware access for GPIO, I2C, UART, and TIM peripherals.
  * **ECU_Abstraction:** Wraps HAL functions into pure hardware-independent communication interfaces (`EcuAbs_I2c`, `EcuAbs_Gpio`, `EcuAbs_Uart`). This layer has no knowledge of specific external devices — it only provides generic read/write/control operations.
  * **Service:** Uses ECU Abstraction interfaces to implement device-specific and system-level services. Includes `Svc_Bme280` (BME280 sensor control via EcuAbs_I2c), `Svc_Log` (error logging via EcuAbs_Uart), and `Svc_StateMgr` (ECU state management). This is the topmost BSW layer that interfaces with the RTE.

## Data Flow

### Upward Path (Sensor → Application)
1. **[BME280]** → I2C → **HAL_I2C (MCAL)** → **EcuAbs_I2c**
   (Raw I2C read/write via hardware-independent interface)
2. **EcuAbs_I2c** → **Svc_Bme280** → **RTE_APP**
   (Service reads BME280 registers and stores raw data in RTE_APP data buffer)
3. **ASW_Sensor** ← reads from **RTE_APP** → **ASW_Diag** → **ASW_Manage**
   (Compensation, filtering, diagnostics, state control)

### Downward Path (Application → Output)
4. **ASW** → writes to **RTE_APP** → **Svc_Log** → **EcuAbs_Uart** → **HAL_UART (MCAL)** → **[UART Terminal]**
   (ASW writes processed data to RTE_APP, Service reads and outputs via UART)
